# Cloud Resume Infra Lab

Cloud infrastructure lab covering AWS and Azure static website deployment, CDN, DNS, SSL, and Terraform-based automation.

This repository is part of my public DevOps portfolio and documents hands-on work around hosting a resume-style web application in the cloud using both manual configuration and Infrastructure as Code approaches. It reflects practical work across cloud services, static site hosting, Terraform, delivery workflows, and deployment architecture.

## Core Features

- AWS and Azure static website deployment
- Terraform-based infrastructure automation
- CDN, DNS, and HTTPS configuration concepts
- Manual versus automated deployment patterns
- Resume-style cloud hosting project structure
- Public infrastructure-focused portfolio work

## Project Scope

This project explores two deployment paths for the same type of application: a static resume website.

The goal is to compare cloud hosting approaches while strengthening practical skills in:
- static website hosting
- DNS and custom domain setup
- CDN configuration
- SSL/TLS enablement
- Infrastructure as Code
- deployment workflow design

## AWS Deployment Path

The AWS side of this project focuses on manually deploying a static website using core AWS services.

### Services Used
- Amazon S3
- Amazon Route 53
- AWS Certificate Manager
- Amazon CloudFront

### What This Path Demonstrates
- Static website hosting with S3
- DNS management with Route 53
- HTTPS certificate handling with ACM
- CDN distribution with CloudFront
- End-to-end manual cloud setup for a public web application

## Azure Deployment Path

The Azure side of this project focuses on deploying the same type of website using Terraform and Azure services.

### Services Used
- Azure Resource Group
- Azure Storage Account
- Azure CDN
- Azure DNS
- Azure CLI
- Terraform

### What This Path Demonstrates
- Infrastructure as Code with Terraform
- Azure static website hosting
- CDN-backed content delivery
- DNS and custom domain planning
- Cloud resource provisioning through repeatable configuration

## Manual vs Automated Approach

One of the main goals of this repository is to compare manual deployment work with automated infrastructure provisioning.

### Manual Approach
Useful for understanding the platform, service relationships, and deployment flow step by step.

### Automated Approach
Useful for repeatability, consistency, change tracking, and infrastructure management at scale.
